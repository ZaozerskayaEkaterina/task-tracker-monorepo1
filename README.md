# Task Tracker Monorepo

Распределённая система управления задачами, разработанная в рамках практического задания. Проект представляет собой бэкенд для сервиса управления задачами, состоящий из двух независимых микросервисов.

## Архитектура

Проект состоит из двух микросервисов, каждый из которых имеет свою базу данных и взаимодействует друг с другом по сети (HTTP):

- **User Service** — управление пользователями (порт `8081`)
- **Task Service** — управление задачами и делегирование (порт `8082`)

## Технологический стек

- **Java 17**
- **Spring Boot 3.x**
- **Spring Data JPA**
- **H2 Database** (in-memory)
- **Liquibase** (миграции БД)
- **SpringDoc OpenAPI** (Swagger UI)
- **Maven**
- **GitLab CI / GitHub Actions**
- **Docker & Docker Compose**

## Функциональные возможности

### Базовый уровень (оценка «Хорошо»)

#### 1. User Service (порт `8081`)
- Сущность **User**: `id`, `firstName`, `lastName`, `email`
- REST API:
  - `POST /users` — создание пользователя
  - `GET /users/{id}` — получение пользователя по ID
- База данных: **H2 (in-memory)**

#### 2. Task Service (порт `8082`)
- Сущность **Task**: `id`, `title`, `status`, `creatorId`, `assigneeId`
- Статусы: `TODO`, `IN_PROGRESS`, `DONE`
- REST API:
  - `POST /tasks` — создание задачи
  - `GET /tasks?userId={userId}` — получение списка задач пользователя
- База данных: **H2 (in-memory)**

#### 3. Механика делегирования
- Эндпоинт: `POST /tasks/{task_id}/delegate?assigneeId=X`
- При вызове `Task Service` отправляет HTTP-запрос в `User Service` для проверки существования пользователя.
- Если пользователь не найден — возвращается ошибка `404 Not Found`.

#### 4. Документация API
- Подключена библиотека `springdoc-openapi-starter-webmvc-ui`
- Swagger UI доступен по адресам:
  - `http://localhost:8081/swagger-ui.html`
  - `http://localhost:8082/swagger-ui.html`

### Продвинутый уровень (оценка «Отлично»)

#### Качество (`*`)
- Написаны unit-тесты для контроллеров (`JUnit 5` + `MockMvc`)
- Настроены миграции БД через **Liquibase**
- Автоматическое создание таблиц через Hibernate **отключено** (`ddl-auto: none`)

#### Инфраструктура (`**`)
- Оба сервиса упакованы в **Docker-контейнеры**
- В корне проекта написан `docker-compose.yml` для запуска всей системы одной командой
- База данных **H2** заменена на **PostgreSQL** (две логические БД в одном контейнере)
- URL соседнего сервиса вынесен в `application.yml` и читается через `@Value`
